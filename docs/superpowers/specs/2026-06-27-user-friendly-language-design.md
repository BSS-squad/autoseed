# User-Friendly Language Design

## Goal

Сделать весь видимый текст AutoSeed понятным обычному игроку Squad без знания внутренней архитектуры сайта.

## Audience

Основной пользователь хочет выбрать сервер, подключиться к нему и посмотреть розыгрыши. Он не обязан знать, что такое exporter, endpoint, snapshot, redirect или Raffle.

## Principles

- Описывать действие или результат, а не технический механизм.
- Использовать короткие русские формулировки в заголовках, статусах, подсказках и ошибках.
- Сохранять знакомые сообществу слова `Автосид`, `сид` и `рассид`.
- Сохранять названия Squad, Steam и серверов.
- Не менять внутренние типы, API-поля, маршруты, test id и имена функций только ради текста.

## Approved Vocabulary

| Current wording | User-facing wording |
| --- | --- |
| `BSS Raffle` | `Розыгрыши BSS` |
| `snapshot`, `снимок` | `данные`, `обновлено` |
| `public snapshot` | `данные серверов` |
| `exporter`, `экспортеры` | `связь с серверами` |
| `endpoint` | убрать из пользовательского текста |
| `Raffles` | `Розыгрыши` |
| `автоконнектор`, `коннектор` | `автоподключение` |
| `служебное окно коннектора` | `окно автоподключения` |
| `цель`, `текущая цель` | `выбранный сервер` |
| `Боевой режим` | `Обычный режим` |
| `UTC+3` | `по московскому времени` |
| `авто` | `запущен автоматически` |
| `ручной` | `запущен администратором` |
| `AutoSeed failed to start` | `Не удалось запустить Автосид` |

## Scope

Замена применяется к:

- главному экрану и навигации;
- странице победителей;
- карточкам серверов и сводке;
- подсказкам и пошаговой инструкции;
- окну автоподключения;
- видимым ошибкам и журналу событий;
- title, meta description и fallback-ошибке запуска.

Технические термины в README и интеграционной документации остаются, если они описывают реальный API-контракт для разработчиков.

## Key Copy

Страница победителей объясняет источник данных без внутренних деталей: «Здесь собраны текущие розыгрыши и история победителей со всех серверов BSS». Empty state сообщает: «Данные о розыгрышах пока не поступили. Загляните позже».

Статус получения данных называется `Обновлено`. Состояние интеграций называется `Связь с серверами`. Выбранный сервер везде называется одинаково и не сокращается до `цель`.

## Verification

- E2E-тесты проверяют новые основные заголовки и статусы.
- Отдельная проверка не допускает видимые `snapshot`, `снимок`, `Raffle`, `exporter`, `экспортер`, `endpoint`, `коннектор`, `цель` и `Боевой`.
- Внутренние поля `snapshot`, `raffles`, `exporters` и API URLs продолжают работать без изменения контракта.
- Desktop и mobile QA проверяют переносы текста, отсутствие переполнения и ошибки консоли.
